## 一、项目总体进展

康健图谱是一个面向运动康复场景的个性化康复处方生成系统。系统围绕"用户评估 -> 智能处方生成 -> 动作跟练反馈 -> 训练记录追踪 -> 医患协同与后台管理"的流程展开，目标是让用户通过 Web 端提交主诉、疼痛部位、活动能力等信息后，获得结构化、可执行、带安全提醒的康复训练建议。

当前项目已完成可运行的 Web MVP，并在 MVP 基础上扩展了用户认证、患者档案、康复处方、训练打卡、姿态纠正、RAG 知识问答、医患协同、后台管理、处方导出和部署配置等模块。后端 API 已形成较完整的业务接口体系，前端已具备问诊、处方展示、训练反馈、档案管理、知识科普和后台管理等页面基础，项目进入中期联调、测试修复和展示材料完善阶段。

## 三、各小组阶段成果

### 1. 架构与后端组

后端采用 FastAPI + SQLAlchemy + SQLite 的开发架构，并预留 PostgreSQL、Redis、MinIO、RAG 向量检索等生产扩展能力。当前后端已实现约 70 个 API 路由，覆盖用户、患者档案、影像报告、训练打卡、处方生成、姿态推理、医患协同、后台管理和知识问答等流程。

已完成成果：

- 建立 `backend/app` 后端模块结构，包含 `api.py`、`models.py`、`schema.py`、`crud.py`、`database.py` 等核心文件。
- 实现用户注册、登录、JWT 鉴权和角色区分，为患者、医生、管理员三类角色提供基础支撑。
- 完成患者档案 CRUD，支持保存姓名、年龄、疼痛部位、既往史、过敏史、手术史和康复目标等信息。
- 完成康复处方生成接口，支持结合用户主诉、疼痛部位、活动评分和知识库动作，生成个性化康复建议。
- 接入 DeepSeek / 火山方舟豆包 OpenAI 兼容接口，并提供本地兜底机制，降低外部模型不可用对演示的影响。
- 实现处方导出、处方历史查询、处方审核、医生调整建议和患者决策等医患协同功能。
- 完成 RAG 知识问答和康复百科接口，能够围绕颈部、肩部、腰部、膝关节、踝关节等部位给出科普建议。
- 补充部署相关能力，包括健康检查、就绪检查、Docker Compose、生产存储栈配置说明等。

后端组当前重点：

- 修复训练打卡与康复进度报告相关测试失败问题。
- 继续校验各接口的权限边界、异常处理和返回结构一致性。
- 整理接口测试截图、典型请求响应和演示脚本。

### 2. 算法与测试组

算法与测试组主要负责动作纠正、姿态关键点处理、评分反馈和系统测试。当前系统已支持基于关键点的动作反馈，能够根据动作 ID、关键点坐标和可见度返回纠正建议、评分和状态。

已完成成果：

- 实现 `POST /api/correct_pose` 姿态纠正接口，支持动作 ID、关键点数组、可见度和时间戳输入。
- 完成颈部侧屈、靠墙静蹲等动作的规则反馈，并持续扩展复杂动作的判断逻辑。
- 实现角度、距离、对称性和关键点可见度等判定方式，增强动作评分的解释性。
- 接入 RTMPose ONNX 推理相关接口，为后续真实视频流关键点识别提供基础。
- 提供 3D 骨骼数据和 AR 叠加接口，支持将姿态关键点转换为更直观的视觉反馈数据。
- 完成语音纠错提示接口，可根据动作状态和评分生成不同语气的训练提示文本。
- 编写并运行后端自动化测试脚本，最新测试覆盖 16 个主流程。

### 3. 前端与交互组

## 五、当前问题与风险

1. 自动化测试仍有 2 项失败，主要集中在训练打卡统计和康复进度报告，需要优先修复。
2. 真实摄像头关键点提取仍在接入阶段，目前演示主要依赖模拟关键点或后端推理接口。
3. 姿态纠正算法虽已可用，但尚未完整覆盖知识库全部动作，后续需要按高频动作优先扩展。
4. 大模型接口依赖外部服务，可能出现网络延迟、鉴权失败或响应格式不稳定，需要保留本地兜底。
5. 当前 SQLite 适合本地演示，若进行多人并发或线上部署，需要切换 PostgreSQL 并完善数据迁移。
6. 项目涉及康复建议，展示时需明确说明系统不能替代医生诊断，只用于课程项目和辅助演示。

3. 总体架构：介绍前端、后端、数据库、知识库、大模型和姿态算法之间的关系。
4. 各小组进度：分别由后端、算法测试、前端组说明本组负责模块和完成情况。
5. 功能演示：按"注册登录 -> 患者档案 -> 问诊输入 -> 处方生成 -> 动作反馈 -> 知识问答/后台管理"流程展示。
6. 测试结果：说明最新测试 16 项通过 14 项，失败 2 项正在修复，体现项目真实进度。
7. 后续计划：突出真实摄像头接入、动作算法扩展、测试修复和答辩材料完善。

## 八、中期总结
