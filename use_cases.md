## 用例详细描述

### UC-1: 患者获取个性化康复处方

**用例编号**：UC-1  
**用例名称**：患者获取个性化康复处方  
**参与者**：患者、LLM系统、知识库  
**前置条件**：
- 患者已注册并登录系统
- 系统知识库已加载

**主流程**：
1. 患者点击"获取康复处方"
2. 系统展示患者信息输入表单
3. 患者输入身体状况信息（年龄、体重、受伤部位等）
4. 患者选择康复需求（运动类型、目标等）
5. 患者确认提交
6. 系统调用LLM生成个性化康复处方
7. 系统展示康复处方（包含动作、次数、频率、安全建议）
8. 患者可选择：查看详情、打印、分享或保存处方

**后置条件**：
- 处方已保存到系统
- 患者可随时查看历史处方

**异常流程**：
- **E1**: 如果患者信息不完整，系统提示需要补充信息
- **E2**: 如果LLM生成超时，系统显示重试选项
- **E3**: 如果知识库为空，系统显示"暂无可用方案"

**约束条件**：
- 处方生成时间应控制在30秒以内
- 处方安全性需要审核

### UC-2: 医生为患者制定康复计划

**用例编号**：UC-2  
**用例名称**：医生为患者制定康复计划  
**参与者**：医生、患者、LLM系统、知识库  
**前置条件**：
- 医生已登录系统
- 患者信息已在系统中

**主流程**：
1. 医生进入"患者管理"模块
2. 医生选择或搜索某个患者
3. 医生点击"制定康复计划"
4. 系统展示患者详细信息
5. 医生输入诊断信息和康复目标
6. 医生选择康复类型和难度等级
7. 医生提交
8. 系统调用LLM生成初始康复方案
9. 医生可以编辑、调整、审核方案
10. 医生确认并保存康复计划
11. 系统通知患者新的康复计划已制定

**后置条件**：
- 康复计划已保存并关联到患者
- 患者可以在应用中查看自己的康复计划
- 医生可以随时修改或创建新版本

**异常流程**：
- **E1**: 如果患者没有完整的医学记录，医生可以手动补充
- **E2**: 如果LLM建议与医学规范冲突，医生可以覆盖
- **E3**: 如果患者信息有矛盾，系统给出警告

**约束条件**：
- 只有具有医生权限的用户能制定正式康复计划
- 康复计划需要专业医学审核

### UC-3: 患者追踪康复进展

**用例编号**：UC-3  
**用例名称**：患者追踪康复进展  
**参与者**：患者、系统  
**前置条件**：
- 患者已登录系统
- 患者已开始康复锻炼并记录

**主流程**：
1. 患者进入"我的康复"模块
2. 患者可以查看当前康复计划
3. 患者记录完成的康复锻炼（日期、完成次数、主观感受）
4. 系统记录锻炼数据
5. 患者查看康复进度统计
6. 系统展示可视化图表（完成率、进度曲线、预期vs实际）
7. 患者可以查看医生的反馈和建议

**后置条件**：
- 康复记录已保存
- 数据用于后续分析和调整

**异常流程**：
- **E1**: 如果患者多天未记录，系统提醒
- **E2**: 如果进度异常，系统提示可能需要就医

**约束条件**：
- 数据应实时更新
- 隐私数据不应被共享给非授权人员

### UC-4: 医生管理患者信息

**用例编号**：UC-4  
**用例名称**：医生管理患者信息  
**参与者**：医生、患者、系统  
**前置条件**：
- 医生已登录系统

**主流程**：
1. 医生进入"患者管理"模块
2. 医生查看患者列表（可按科室、状态等筛选）
3. 医生选择患者查看详细信息
4. 医生可以：
   - 查看患者基本信息
   - 查看康复历史
   - 查看锻炼记录
   - 查看康复效果评估
5. 医生可以添加备注和更新患者状态
6. 医生可以导出患者报告

**后置条件**：
- 患者信息已更新
- 医生可随时查看患者最新状态

**异常流程**：
- **E1**: 如果患者数据不完整，医生可以补充
- **E2**: 如果出现数据冲突，系统提示

**约束条件**：
- 医生只能看到自己负责的患者
- 数据访问需要记录审计日志

### UC-5: 管理员维护康复动作知识库

**用例编号**：UC-5  
**用例名称**：管理员维护康复动作知识库  
**参与者**：管理员、系统、知识库  
**前置条件**：
- 管理员已登录系统
- 具有管理员权限

**主流程**：
1. 管理员进入"知识库管理"模块
2. 管理员查看现有康复动作列表
3. 管理员可以：
   - **添加新动作**：输入动作名称、类型、难度、步骤、禁忌等
   - **编辑动作**：修改动作信息
   - **删除动作**：下架过时或不安全的动作
   - **批量导入**：从文件导入动作库
4. 管理员可以管理分类标签
5. 管理员可以管理提示词模板
6. 系统验证数据完整性和合理性
7. 管理员确认保存

**后置条件**：
- 知识库已更新
- 更新会影响新生成的康复方案

**异常流程**：
- **E1**: 如果数据格式不正确，系统提示错误
- **E2**: 如果删除的动作被方案引用，系统提示风险

**约束条件**：
- 只有管理员能修改知识库
- 所有修改需要记录版本历史

### UC-6: LLM生成个性化康复方案

**用例编号**：UC-6  
**用例名称**：LLM生成个性化康复方案  
**参与者**：LLM模型、知识库、医学数据库  
**前置条件**：
- LLM模型已加载
- 知识库包含足够的康复动作数据
- 患者信息已输入

**主流程**：
1. 系统准备提示词（基于患者信息和医学知识）
2. 系统调用LLM模型进行推理
3. LLM基于患者情况生成康复方案
4. LLM返回方案建议（包含动作、频率、强度等）
5. 系统对返回结果进行验证（安全性、完整性）
6. 系统展示结果给医生或患者

**后置条件**：
- 康复方案已生成并可供使用

**异常流程**：
- **E1**: 如果LLM输出不符合医学规范，进行修复或重试
- **E2**: 如果超时，显示备选方案或让医生手动制定

**约束条件**：
- 方案必须符合医学安全标准
- 生成时间应控制在合理范围内

### UC-7: 患者查看和下载康复教育资源

**用例编号**：UC-7  
**用例名称**：患者查看和下载康复教育资源  
**参与者**：患者、系统、资源库  
**前置条件**：
- 患者已登录系统
- 系统包含教育资源

**主流程**：
1. 患者进入"康复学院"或"学习资源"模块
2. 患者浏览资源分类（按受伤部位、康复阶段等）
3. 患者选择某个资源查看
4. 系统展示资源详情（文章、视频、图解等）
5. 患者可以：
   - 收藏资源
   - 下载资源
   - 分享资源
   - 评分反馈

**后置条件**：
- 患者已获取学习资源
- 系统记录了资源使用数据

**异常流程**：
- **E1**: 如果资源不可用，显示替代资源

### UC-8: 医生导出和分享康复方案

**用例编号**：UC-8  
**用例名称**：医生导出和分享康复方案  
**参与者**：医生、患者、系统  
**前置条件**：
- 医生已登录
- 康复方案已生成

**主流程**：
1. 医生选择要导出的康复方案
2. 医生选择导出格式（PDF、Word、Excel等）
3. 医生可以自定义导出内容（是否包含医学建议、安全提示等）
4. 系统生成导出文件
5. 医生可以：
   - 下载文件
   - 直接打印
   - 通过链接分享给患者

**后置条件**：
- 方案文件已生成
- 患者可以通过多种渠道获取

---

### UC-9: 系统生成康复效果分析报告

**用例编号**：UC-9  
**用例名称**：系统生成康复效果分析报告  
**参与者**：医疗管理者、系统、数据分析模块  
**前置条件**：
- 系统已积累足够的患者数据
- 具有分析权限

**主流程**：
1. 管理者进入"数据分析"模块
2. 管理者选择分析时间范围和维度
3. 系统收集和处理数据
4. 系统生成可视化报告包括：
   - 总患者数、案例数
   - 各康复类型的分布
   - 平均康复周期
   - 康复成功率
   - 常见问题分析
5. 管理者可以导出报告

**后置条件**：
- 报告已生成，可用于管理决策
