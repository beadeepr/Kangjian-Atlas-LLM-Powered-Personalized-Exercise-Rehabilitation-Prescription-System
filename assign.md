## 当前功能完成情况

### 已完成

- ✅ 项目初始化
  - 建立 `backend`、`frontend`、`knowledge` 项目结构。
  - 完成 FastAPI 后端基础入口。
  - 完成前端静态页面、样式和交互逻辑。
  - 完成动作知识库和提示词模板目录。

- ✅ 系统架构与后端服务
  - 使用 FastAPI 实现 REST API 服务。
  - 完成 CORS 配置和健康检查接口 `/health`。
  - 完成 SQLite + SQLAlchemy 数据持久化。
  - 实现用户、处方、处方动作、姿态反馈四类数据库模型。
  - 支持启动时初始化数据库，并对开发阶段旧表自动补充 `raw_response` 和 `user_id` 字段。

- ✅ 用户认证与处方数据隔离
  - 完成用户注册接口 `POST /api/register`。
  - 完成用户登录接口 `POST /api/login`。
  - 使用 JWT 进行身份认证，支持前端保存登录态并携带令牌访问接口。
  - 处方生成记录与当前登录用户绑定。
  - 历史处方查询按当前用户过滤，避免不同用户数据混淆。

- ✅ 康复处方生成 API
  - 完成 `POST /api/generate_prescription`。
  - 支持用户姓名、年龄、主诉、既往病史、疼痛部位、活动度评分输入。
  - 根据症状和疼痛部位从知识库检索候选动作。
  - 返回处方摘要、动作列表、频次、禁忌症、进阶和降阶建议。
  - 保存处方记录、动作记录和模型原始响应。
  - 支持登录用户维度的处方历史追踪。

- ✅ 大模型集成
  - 完成豆包/火山方舟 API 接入。
  - 支持从 `.env` 自动读取 `DOUBAO_API_KEY`、`DOUBAO_BASE_URL`、`DOUBAO_MODEL_ID`。
  - 实现 OpenAI 兼容 `chat/completions` 调用方式。
  - 支持模型 JSON 输出解析。
  - 对敏感字段进行脱敏处理。
  - 大模型调用失败时提供本地兜底摘要。
  - 完成调试接口 `POST /api/test_doubao`。

- ✅ 康复知识库
  - 扩展 `knowledge/actions.json`，覆盖颈部、肩部、腰部、膝关节等常见康复场景。
  - 每个动作包含动作 ID、适用病症、身体部位、组数、次数、频次、动作描述、禁忌症、进阶规则和降阶方案。
  - 当前知识库包含 14 个动作。

- ✅ 提示词模板
  - 完成 `knowledge/prompt_template.txt`。
  - 将用户输入和知识库候选动作组织为上下文。
  - 明确安全要求：不替代医生诊断、不编造动作、出现危险信号时建议就医。
  - 明确字段优先级：疼痛部位冲突时以用户勾选的疼痛部位作为动作选择依据，主诉用于描述症状细节，活动度仅影响强度表述。
  - 要求模型输出结构化 JSON，便于后端解析。

- ✅ 姿态纠正 API
  - 完成 `POST /api/correct_pose`。
  - 接收 `action_id`、`keypoints`、`visibility`、`timestamp`。
  - 返回 `feedback`、`score`、`status`。
  - 当前支持 `neck_side_bend`、`wall_squat` 等动作算法，并持续扩展复杂动作规则。
  - 优化角度、距离和关键点可见度判断，增强反馈信息和评分逻辑。
  - 姿态反馈记录会写入 SQLite。

- ✅ 前端基础演示
  - 支持用户问诊表单。
  - 支持疼痛部位选择和活动度评分。
  - 支持处方展示和动作卡片。
  - 支持 Demo 模式和真实后端 API 模式切换。
  - 支持模拟姿态检测并展示反馈、分数和状态。

## 风险与问题

- 大模型接口依赖外部服务，可能出现网络延迟、鉴权失败或响应格式不稳定。
- 当前动作纠正算法覆盖动作有限，尚未覆盖知识库中的所有动作。
- 前端真实视频关键点提取尚未接入 MediaPipe，目前主要通过模拟关键点验证后端接口。
- GitHub 访问依赖本地网络和代理环境，推送或拉取时可能出现连接失败。
- SQLite 适合 MVP 和本地演示，正式部署需要规划数据库迁移和备份。
- 康复建议仅用于学习和辅助演示，不能替代医生面诊和专业康复治疗。
