## 一、项目背景

### 1.1 行业现状与痛点

运动损伤康复和慢性病管理是现代社会的重大健康挑战。随着久坐办公人群增加，颈椎病、腰椎间盘突出等疾病的发病率持续上升。传统康复治疗存在以下核心痛点：

| 问题维度 | 具体表现 |
|--------|--------|
| **专业资源稀缺** | 康复医师数量严重不足，患者每周仅能获得1-2次面对面指导 |
| **居家康复缺乏依据** | 患者出院后缺乏科学指导，要么运动不足导致功能恢复慢，要么运动过量导致二次损伤 |
| **康复方案同质化** | 传统康复计划多为通用模板，难以针对个体差异（年龄、体质、病程）进行个性化调整 |
| **依从性监测困难** | 患者居家训练质量无法保证，动作不规范难以被发现，日积月累形成代偿问题 |
| **康复周期漫长** | 慢性病康复往往需要数月持续干预，患者容易中途放弃 |

### 1.2 政策与市场机遇

- **政策利好**：《"健康中国2030"规划纲要》明确提出发展康复医学，国家卫健委推动三级康复医疗服务体系建设
- **市场规模**：2025年中国康复医疗器械市场规模超过1500亿元，居家康复设备增速显著
- **技术成熟**：姿态估计算法（HRNet、RTMPose）的精度已达医疗级水准，多模态大模型（Qwen2.5-VL、InternVL3）实现视频理解与自然语言生成的深度融合
- **数据基础**：Rehab-Pile（2025年发布）等标准化康复动作数据集的涌现，为AI康复处方提供训练基础

### 1.3 项目价值

本项目致力于打通"用户评估→智能处方生成→动作示范→实时纠正→效果追踪"的完整康复闭环，让运动损伤和慢性病患者在家也能获得专业级的康复指导，实现"将康复治疗师装进手机"的愿景。

## 四、功能详细分析

### 4.1 M2 伤病评估模块

#### 功能描述
全面的用户伤病信息采集是生成精准康复处方的前提。该模块通过结构化问卷、自测引导和影像报告分析，构建完整的用户健康画像。

#### 详细功能点

**结构化问卷**
- 伤病类型选择（拉伤/扭伤/劳损/骨折术后/关节退变等）
- 疼痛部位标注（人体图交互选择，支持多部位）
- 疼痛性质描述（钝痛/刺痛/灼痛/酸胀）
- 病程时长与近期变化趋势
- 既往治疗经历（理疗/手术/药物）

**活动度自测**
- 标准动作示范视频（如颈椎前屈/后伸/侧屈角度测量）
- 文字+图示说明自测方法
- 用户输入测量结果，系统自动评估受限程度

**影像报告OCR**
- 支持上传X光/MRI/CT报告图片
- 使用OCR（InternVL3）提取关键诊断信息
- 自动关联相关伤病知识

**风险预警**
- 检测到红旗症状（red flags）时强制提示就医
- 如：持续剧痛、麻木放射、体重骤降等

#### 数据流
```
用户录入 → 问卷数据整合 → 风险评估
→ [高风险] → 提示就医建议
→ [低风险] → 进入处方生成流程
```

### 4.2 M3 康复处方生成模块（核心模块）

#### 功能描述
这是整个系统的AI核心能力，基于用户评估结果和康复医学知识库，自动生成个性化、可执行的康复处方。

#### 技术实现路径

**方案A（推荐）：RAG + LLM 处方生成**
- 用户评估数据 → 结构化特征提取
- 从康复知识图谱检索相关处方规则、动作库
- LLM综合推理，生成个性化处方
- 输出：动作列表 + 训练参数 + 进阶策略

**处方结构设计**
```json
{
  "prescription_id": "RX-20260518-001",
  "user_id": "user_xxx",
  "primary_diagnosis": "颈椎生理曲度变直伴颈型颈椎病",
  "target_level": "缓解疼痛、恢复活动度",
  "duration": "4周",
  "frequency": "每日2次",
  "exercises": [
    {
      "id": "EX-001",
      "name": "颈椎米字操",
      "category": "活动度训练",
      "difficulty": "初级",
      "duration": "5分钟",
      "repetitions": "每个方向10次",
      "frequency": "每日2次",
      "contraindications": ["颈椎急性疼痛期禁用"],
      "demo_media": "video/demo_001.mp4"
    }
  ],
  "progression_plan": [
    {"week": "1-2", "focus": "疼痛缓解+轻度活动"},
    {"week": "3-4", "focus": "活动度恢复+力量训练"}
  ],
  "warnings": ["训练中出现剧痛立即停止"],
  "generated_by": "AI辅助生成，仅供参考"]
}
```

#### 知识库构建

使用 **Rehab-Pile（法国上阿尔萨斯大学等，2025年发布）** 作为动作分类基础：
- 60个细粒度康复任务（39分类+21回归）
- 覆盖颈椎、肩周、腰椎、膝关节等常见伤病类型
- 配套临床康复指南映射

使用 `LangChain + Qdrant` 构建RAG检索引擎：
- 康复医学教科书条文
- 临床操作指南（CPG）
- 专家共识文档
- 禁忌症数据库

### 4.3 M4 康复动作库模块

#### 功能描述
标准化的康复动作库是处方的执行载体，每个动作都配有详细的图文/视频演示和动作要领说明。

#### 动作库结构

| 动作分类 | 示例动作 | 适用场景 |
|--------|--------|--------|
| **活动度训练** | 颈椎米字操、肩环绕、腰部扭转 | 关节僵硬、活动受限 |
| **力量训练** | 颈后肌群抗阻、臀桥、死虫式 | 肌肉萎缩、稳定性下降 |
| **拉伸训练** | 胸大肌拉伸、腘绳肌拉伸 | 肌肉紧张、姿势异常 |
| **平衡训练** | 单腿站立、平衡垫训练 | 本体感觉减退、跌倒风险 |
| **神经松动** | 颈椎神经松动术、坐骨神经松动 | 神经症状（麻木/刺痛） |

#### 动作详情页设计
- 高清演示视频（正面+侧面双视角）
- 分步图解（准备姿势→动作执行→还原）
- 动作要领文字说明（呼吸配合、发力感）
- 常见错误对比图
- 禁忌症提示
- 用户反馈入口（动作执行难度评分）

### 4.4 M5 实时动作跟练模块（核心模块）

#### 功能描述
通过设备摄像头实时捕捉用户动作，结合姿态估计算法进行规范性评估和即时纠错，是系统最具技术亮点的功能。

#### 技术实现

**姿态估计模型选型**

| 模型 | 精度 | 速度 | 推荐场景 |
|-----|------|------|--------|
| RTMPose (ONNX) | 高 | 实时(30+ FPS) | **推荐**：边缘设备友好 |
| MediaPipe Pose | 中 | 实时 | Web端快速集成 |
| HRNet | 最高 | 较慢 | 高精度需求场景 |

**推荐方案（双模型协同）**：
```
摄像头视频流 → MediaPipe/摄像头API实时预览（前端）
             ↓
         关键帧截取 → RTMPose姿态估计（后端）
             ↓
         17个关键点坐标 → 动作评分算法
             ↓
         规范性评分 + 纠错建议 → 实时反馈
```

**动作评分算法**
- 计算关键角度（如肘关节角度）
- 与标准动作角度范围对比
- 输出：合格/角度偏差/幅度不足
- 错误类型分类（如"右肩抬起过高"）

**实时反馈设计**
- 文字提示："右肩略微上抬，请保持与左肩平齐"
- 语音播报（可开关）
- 正确姿势影子叠加（AR效果）
- 训练完成统计（已做X组，剩余Y组）

#### 技术难点应对

1. **光线变化**：使用图像增强预处理，适应不同环境光
2. **遮挡问题**：多视角融合或提示"请确保全身可见"
3. **边缘设备性能**：前端轻量级预览 + 后端精确实时处理
4. **隐私保护**：视频不上传服务器，本地实时处理（WebRTC/WebGL）

- 综合疼痛、活动度、训练数据

- 异常预警（如疼痛指数持续上升）

## 五、重难点功能注意

### 5.1 🔴 重点难题：动作姿态识别的准确性

**问题描述**  
实时动作纠正依赖精确的人体关键点检测，但用户拍摄角度、光线条件、穿着差异都会影响识别准确率。

**解决策略**
1. **多关键点融合**：使用RTMPose检测17个COCO标准关键点，交叉验证提高鲁棒性
2. **角度容错设计**：设置合理阈值（如±15°），避免过度敏感导致频繁误报
3. **渐进式提示**：先从"大方向"提示（如"动作幅度不够"），再细化到具体关节
4. **前端预览过滤**：前端MediaPipe实时预览，显著偏离才调用后端精确检测
5. **用户校准环节**：首次使用时要求用户完成标准站姿校准

---

### 5.2 🔴 重点难题：处方生成的专业性与安全性

**问题描述**  
康复处方直接涉及用户身体健康，错误处方可能导致二次损伤，系统面临法律风险。

**解决策略**
1. **禁忌症知识库**：构建完整的动作-伤病禁忌映射表，系统自动过滤高风险动作
2. **分级风险控制**：
   - 绿色（普通建议）：通用保健动作，任何人均可执行
   - 黄色（需注意）：需用户自评后执行，系统提示风险
   - 红色（需确认）：需用户提供医疗证明或医生确认
3. **红旗症状拦截**：检测到高风险信号时强制提示就医，不生成处方
4. **免责声明机制**：所有处方显著位置标注"仅供参考，请遵医嘱"
5. **专家校验集**：建立100+标准测试用例，定期评估处方安全性

---

### 5.3 🔴 重点难题：小样本数据下的处方个性化

**问题描述**  
康复处方的个性化需要大量临床数据支撑，但康复医学领域数据集相对有限。

**解决策略**
1. **知识图谱增强**：以Rehab-Pile数据集为基础，叠加人工整理的临床指南，构建结构化知识图谱
2. **规则+LLM混合**：基础处方由规则引擎生成，个性化调整由LLM完成，平衡可控性与灵活性
3. **用户反馈闭环**：收集用户执行反馈，持续优化处方推荐算法
4. **保守策略优先**：数据不足时采用更保守的处方参数，确保安全

---

### 5.4 🟡 难点：多伤病叠加的处方协调

**问题描述**  
用户可能同时存在多个伤病部位（如颈椎病+肩周炎+腰椎病），处方需协调多个康复目标。

**解决策略**
1. **优先级排序**：根据用户自评和评估结果，确定主要康复目标和次要目标
2. **动作兼容分析**：检测不同伤病处方中动作的冲突性（如某些动作可能加重某一部位）
3. **分时训练方案**：将不同部位的康复动作分配到不同时段，避免相互干扰
4. **增量叠加策略**：先稳定一个部位，再逐步加入其他部位的训练

---

### 5.5 🟡 难点：实时跟练的延迟优化

**问题描述**  
实时动作反馈需要在低延迟下完成（<200ms），否则用户体验极差。

**解决策略**
1. **边缘计算**：前端MediaPipe/JavaScript实时预览，不依赖后端
2. **异步批处理**：后端RTMPose采用异步处理，每秒处理10-15帧即可
3. **缓存优化**：动作标准参数预加载，避免每次请求重新计算
4. **降级策略**：网络慢时自动切换为"录屏后分析"模式

---

### 5.6 🟡 难点：隐私保护与数据安全

**问题描述**  
用户需要开启摄像头进行动作跟练，视频数据涉及隐私，需严格保护。

**解决策略**
1. **本地处理优先**：所有视频帧在用户设备本地处理，不上传服务器
2. **仅上传关键数据**：姿态坐标序列而非完整视频，大幅减少数据传输
3. **数据加密存储**：用户授权的历史数据加密存储，支持随时删除
4. **透明隐私政策**：明确告知数据用途，获取用户知情同意

## 六、亮点和特色功能

### ✨ 亮点1：AI驱动的个性化处方生成

这是本项目最核心的技术创新。系统不是简单地从动作库里挑选动作，而是根据用户评估数据，通过RAG知识检索+LLM推理，生成真正个性化的康复方案：

- 传统方案：通用康复动作视频，用户自己判断是否适合
- **本系统**：用户评估 → 知识匹配 → 个性化适配 → 专属处方

例如：同样是颈椎病患者，30岁程序员（长期伏案、肌肉紧张型）与60岁退休老人（退变型、骨质疏松倾向）会得到完全不同的处方：前者侧重肌肉松解和姿势矫正，后者侧重安全范围内的活动度维持和力量训练。

---

### ✨ 亮点2：实时动作跟练与智能纠错

通过摄像头实时捕捉用户动作，结合姿态估计算法进行规范性评估和即时纠错：

- **视觉反馈**：正确姿势影子叠加，直观展示标准动作
- **语音提示**：解放双手的语音纠错（如"动作很好，请保持"）
- **多维度评分**：角度准确性、完成度、肌肉发力感等多维度评估
- **训练游戏化**：设置"动作挑战赛"等激励机制，提升依从性

---

### ✨ 亮点3：康复效果可视化追踪

创新的"康复进度仪表盘"：

- 疼痛指数变化曲线（VAS追踪）
- 活动度改善百分比
- 训练完成率统计
- AI周评估报告（基于数据趋势的智能分析）
- 里程碑成就（连续打卡奖励、阶段性进步表彰）

让康复过程"看得见"，帮助用户建立信心，提高依从性。

---

### ✨ 亮点4：多模态康复指导（视频+图文+AR）

丰富的动作示范形式：

| 形式 | 特点 | 适用场景 |
|-----|------|--------|
| 高清视频演示 | 双视角（正面+侧面），可慢放 | 首次学习动作 |
| 分步图解 | 静态图片+文字要点 | 快速回顾 |
| 3D骨骼动画 | 人体骨骼模型演示 | 理解发力原理 |
| AR叠加 | 真实场景+虚拟指导 | 跟练时实时对照 |

---

### ✨ 亮点5：智能处方动态调整

基于用户反馈的持续优化机制：

- **周期性评估**：每两周提示用户进行自评
- **AI自动调整**：若训练完成率低，自动降低强度；若效果不佳，推荐增加训练频次
- **异常预警**：疼痛指数异常上升时建议就医
- **医生协同**（可选）：用户可将处方分享给康复医生，获取专业意见

### 7.2 AI模型选型详细说明

#### 姿态估计模型选型对比

| 模型 | 精度 | 速度 | 部署成本 | 推荐度 |
