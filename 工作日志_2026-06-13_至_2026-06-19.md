# 工作日志

## 项目名称
康健图谱 - 基于大模型的个性化运动康复处方生成系统

## 时间范围
2026年6月13日 - 2026年6月19日

## 个人职责
后端开发、豆包大模型集成、处方生成逻辑、提示词模板维护、前后端联调支持。

---

## 2026年6月13日

### 今日工作
1. 完成 JWT 登录能力的后端与前端联调，支持用户注册、登录和身份认证。
2. 将康复处方数据与登录用户进行绑定，避免不同用户之间处方记录混淆。
3. 配合合并远程分支代码，处理多人协作过程中的分支同步问题。
4. 优化部分复杂动作的姿态纠正算法，增强反馈信息和评分逻辑。

### 遇到问题
多人协作时本地分支与远程分支存在差异，需要先拉取远程更新再合并本地提交。

### 解决情况
通过 Git 分支同步和合并处理，使本地代码与远程主分支保持一致。

---

## 2026年6月16日

### 今日工作
1. 进行前后端联调，重点验证康复处方生成流程是否能从问诊表单顺利进入结果展示页面。
2. 修复 CORS 配置相关问题，保证前端页面可以正常请求后端接口。
3. 优化跟练流程中的接口调用逻辑，提升处方生成和动作反馈之间的衔接体验。
4. 检查豆包大模型调用链路，确保模型异常时仍可使用本地兜底结果完成演示。

### 遇到问题
前端请求后端接口时存在跨域限制，影响真实 API 模式下的联调。

### 解决情况
调整后端 CORS 配置，并结合前端真实后端模式进行验证。

## 2026年6月17日

### 解决情况

## 2026年6月18日

### 今日工作
1. 根据项目评审反馈，更新 `knowledge/prompt_template.txt` 提示词模板。
2. 新增规则：疼痛部位冲突时，以用户勾选的疼痛部位为动作选择依据；主诉用于描述症状细节；活动度仅影响强度表述。
3. 检查提示词输出格式，继续要求模型只输出 JSON，方便后端解析。
4. 确认 GitHub 远程仓库状态，处理推送和网络代理相关问题。

### 遇到问题
用户主诉、疼痛部位和活动度评分之间可能存在解释冲突，容易导致模型选错动作。

### 解决情况
通过提示词明确字段优先级，降低模型理解偏差，提高处方动作选择的一致性。
